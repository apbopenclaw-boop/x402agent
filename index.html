How It Works

Three steps. No signup, no API keys, no vendor lock-in.

1

Request

Your AI agent sends a standard HTTP request to any paid endpoint.

2

Pay

The server returns HTTP 402. Your agent signs a USDC micropayment on Base.

3

Receive

Re-send with the payment header — structured data returned instantly.

# 1. Agent requests data $ curl https://norwegian-companies-house.fly.dev/company/923609016 # → HTTP 402 Payment Required # → payment-required: {scheme, network, amount, payTo} # 2. Agent signs payment, retries with header # → HTTP 200 OK {"name":"EQUINOR ASA","type":"ASA","employees":21376,...}

Frequently Asked Questions

What is x402agent?▼

x402agent.no is Norway's AI agent API gateway — a collection of Norwegian data APIs that any x402-compatible AI agent can pay for and use instantly. No API keys, no accounts, no subscriptions.

What is the x402 payment protocol?▼

x402 is an open protocol that lets AI agents pay for API access using cryptocurrency micropayments. When an agent hits a paid endpoint, it receives an HTTP 402 response. The agent signs a USDC payment on Base and resends — all in one request cycle.

How much do the APIs cost?▼

Pay-per-query from $0.005. Each endpoint price is in the 402 response. USDC on Base with sub-cent gas fees. No minimums, no commitments.

Do I need an API key?▼

No. All APIs are completely keyless. Any AI agent with a USDC wallet on Base can start immediately. Free endpoints like /health require nothing at all.

Where does the data come from?▼

Official Norwegian sources: Brønnøysundregistrene (companies), SSB (economics), Kartverket (mapping), Patentstyret (IP), MET Norway (weather), Entur (transport), and Common Crawl (SEO).

Which AI agents work with x402?▼

Any agent that can sign EIP-712 typed data. Coinbase AgentKit, Langchain with x402 middleware, and custom agents using x402 Python or TypeScript SDKs all work out of the box.
